# 质量检查指南 v2.0 (生产级)

> **版本**: 2.0.0 | **标准**: ISTQB + IEEE 829 + CMMI DEV 3.0 | **用途**: 全流程质量保障

---

## 概述

本指南定义了 testcase-generator 五阶段流水线的**完整质量保障体系**。每个阶段完成后，必须执行对应的质量门禁检查。

### 质量体系架构

```
┌─────────────────────────────────────────────────────────────┐
│                    质量保障体系                              │
│                                                             │
│  ┌───────────┐  ┌───────────┐  ┌───────────┐              │
│  │ Phase Gate│  │  Metrics  │  │ Reporting │              │
│  │ 门禁系统   │  │ 度量指标   │  │ 报告机制   │              │
│  └─────┬─────┘  └─────┬─────┘  └─────┬─────┘              │
│        │              │              │                     │
│        ▼              ▼              ▼                     │
│  ┌─────────────────────────────────────────────┐           │
│  │          Quality Dashboard                   │           │
│  │  · 实时质量评分     · 缺陷趋势    · 覆盖热力图  │           │
│  └─────────────────────────────────────────────┘           │
└─────────────────────────────────────────────────────────────┘
```

### 核心原则

1. **门禁驱动**：每阶段必须通过质量门禁才能进入下一阶段
2. **量化度量**：所有质量维度都有可测量的数值指标
3. **主动防御**：在问题产生时就检测和修复（而非事后补救）
4. **持续改进**：基于历史数据不断优化质量基线
5. **可追溯性**：每个质量发现都可追溯到具体产物和决策点

## 质量评分计算

### 综合评分公式

```
Phase_Score = Σ(Check_Item_Weight × Item_Pass_Status) / Σ(All_Weights)

其中:
  Item_Pass_Status = 1.0 (Pass)
                  = 0.7 (Pass with Warning)
                  = 0.3 (Fail with Workaround)
                  = 0.0 (Fail)
```

### 评分等级定义

| 等级 | 分数范围 | 含义 | 行动 |
|------|---------|------|------|
| 🟢 **A - 优秀** | 90-100 | 超出预期的质量 | 可直接进入下一阶段 |
| 🟡 **B - 良好** | 80-89 | 达到质量标准，有小瑕疵 | 修复 Warning 后进入下一阶段 |
| 🟠 **C - 合格** | 70-79 | 基本合格，需关注改进点 | 修复所有 Warning 和 Minor Issue |
| 🔴 **D - 不合格** | < 70 | 不满足最低质量要求 | 必须修复后重新审核 |

### 项目整体质量评分

```
Overall_Quality = 
  0.20 × Phase1_Score +
  0.20 × Phase2_Score +
  0.20 × Phase3_Score +
  0.20 × Phase4_Score +
  0.20 × Phase5_Score
```
